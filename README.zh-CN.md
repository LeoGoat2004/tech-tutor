# Tech Tutor

> *你的苏格拉底式学习伙伴，深度理解技术。*

---

## 快速开始

```bash
npx skills add https://github.com/LeoGoat2004/tech-tutor
```

安装后，直接说：*"我想学习 [某个技术]"* 或 *"帮我理解这个项目"*。

---

## 为什么需要它

AI 工具随处可见——写代码、调 bug、设计架构、秒回问题。方便、上瘾，但有一个隐性代价：**人们停止了思考。** 你逐渐失去了独立推理的能力，不再追问"为什么"，不再关心设计决策，最终变成了一个自己都搞不懂的工具的依赖者。

Tech Tutor 的存在就是为了对抗这个趋势。不是拒绝 AI，而是用不同的方式使用它——**构建真正的理解**，而不是生成用完即弃的答案。

目标很简单：学习结束时，你能在没有 AI 辅助的情况下，把学到的东西讲清楚。

---

## 它做什么

一个苏格拉底式的学习伙伴。它不讲课——它提问，挑战你的假设，帮助你自己发现洞见。

**两种学习模式：**

| 模式 | 用途 | 示例 |
|------|------|------|
| **方向探索** | 从零学习一个概念或技术 | *"教我 Transformer 是怎么工作的"* |
| **项目拆解** | 深入理解一个现有代码库 | *"帮我梳理一下这个项目的架构"* |

**项目拆解采用六层递进框架**——从鸟瞰到动手，系统有条理：

```
1. 鸟瞰全景    →  技术栈、目录结构、项目定位
2. 定位入口    →  程序起点、启动流程、初始化顺序
3. 核心模块    →  关键组件、职责划分、依赖关系
4. 核心流程    →  追踪一条完整链路，每环节输入输出
5. 设计理念    →  为什么这么设计，有哪些 trade-off
6. 拓展改动    →  动手改点东西，加个小功能
```

**预置领域：** 前端 · 后端 · AI 基础设施 · Agent 框架 · 多智能体 · 强化学习 · ROS

> 没看到你想学的领域？Tech Tutor 可以通过实时搜索权威资料教授任何 CS/AI 主题。

## 设计哲学：苏格拉底方法

> *"我不能教给任何人任何东西，我只能让他们思考。"* —— 苏格拉底

- **提问，而非告知。** 用问题引导你得出自己的结论。
- **质疑假设。** *"我们在这里假设了什么？如果假设不成立呢？"*
- **深挖本质。** *"你为什么这么认为？"* —— 不停留在第一个答案。
- **承认无知。** 遇到真正未知的东西，坦诚说出来，一起探索。
- **做中学。** 理解来自实践，而非被动阅读。

**核心原则：理解 > 进度。** 覆盖多少内容不重要，真正懂了多少才重要。

## 它怎么工作

```
1. 明确目标     →  学什么？你的水平？记录存在哪？
2. 准备背景     →  搜集权威资料 + 建立心智模型 + 对齐预期
3. 共同学       →  苏格拉底式对话，先全局后细节，节奏你说了算
4. 学习记录     →  自动保存——你的洞见、问题、模糊点
5. 额外成果     →  可选——笔记、卡片、导图等（你要才生成，位置你定）
```

### 双层学习记录体系

```
~/.tech-tutor/                     用户级（跨项目通用）
├── index.yml                      会话索引 + 复习时间
├── project-index.yml              所有项目级记录的全局索引
└── sessions/20260713_transformer/
    ├── record.md                  你的学习轨迹
    └── outputs/                   可选的额外成果物

<项目根目录>/.tech-tutor/          项目级（针对单个项目）
├── index.yml
└── sessions/20260715_langgraph/
    ├── record.md
    └── outputs/
```

记录的是*你的*学习轨迹——恍然大悟的时刻、还模糊的地方、产生的疑问——而不是教科书式的总结。

### 复习与间隔重复

轻量、对话式、没有考试：

- **时间安排**：约 1 / 3 / 7 / 14 / 30 天
- **自主调节**：你自己判断掌握程度，间隔随之调整
- **自然触发**：每次开始新会话时随口一提——可以接受也可以跳过

### 关联学习

当新主题和过去学过的内容有真正的关联时，会自然地提出来。不强行凑关系。
